No trust required.
Just code.

PasaBuy puts the pasabuy transaction on-chain. USDC is locked at posting, released only at confirmed delivery — with an on-chain dispute path if things go wrong.

01
Buyer posts an order
USDC is locked into a Soroban smart contract. Nobody can touch it until delivery is confirmed.
02
Agent accepts
An abroad agent commits to buying and shipping the item. Their identity is recorded on-chain.
03
Agent marks shipped
Item is on its way. A shipping record is stored on-chain. The buyer is notified.
04
Buyer confirms delivery
USDC is released to the agent. Transaction complete — both sides protected the whole time.
Contract State Machine
Open → Accepted → Shipped → Completed
Shipped can also go to → Disputed → Resolved
Contract Functions
FunctionDescription
initialize(admin)Set contract admin for dispute resolution
create_order(...)Buyer locks USDC, creates order
accept_order(agent, id)Agent commits to fulfilling the order
mark_shipped(agent, id)Agent marks item as shipped
confirm_delivery(buyer, id)Buyer confirms receipt, releases USDC
dispute(buyer, id)Buyer freezes funds pending resolution
resolve_dispute(admin, id, refund)Admin sends funds to buyer or agent
